Incident Management

Incident management coordinates the response to unplanned disruptions of an IT service. Its goal is to restore normal service operation as quickly as possible while minimising business impact. In ServiceNow ITSM, incidents are tracked on the incident table and progressed through states from New to Resolved and Closed.

Assignment and routing

Use assignment rules to automatically route incidents to the correct assignment group based on category, configuration item, or location. Combine assignment rules with on-call scheduling so high-priority incidents reach an available engineer without manual triage.

  • Set the impact and urgency so priority is derived consistently.
  • Link related problems to capture root-cause work separately from restoration.
  • Record major incidents with the major incident workflow for coordinated response.

Automating with Flow Designer

Flow Designer provides no-code automation for the incident lifecycle. Build flows and subflows to notify stakeholders, create tasks, and update records as an incident progresses, reducing manual effort and ensuring consistent handling. 

var gr = new GlideRecord('incident');
gr.addQuery('active', true);
gr.addQuery('priority', 1);
gr.query();
while (gr.next()) {
  gs.info('Active P1 incident: ' + gr.getValue('number'));
}

Reporting

Track operational health with metrics such as mean time to resolution (MTTR), reopen rate, and first-call resolution. Dashboards built on these metrics help service owners spot trends and target improvements.
